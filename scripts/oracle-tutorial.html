p3pp4 na Oracle Cloud

Capitulo 1
Visao Geral
O p3pp4 e um solver automatico de simuladores de fisica que utiliza Playwright (browser headless), Python, Node.js e a IA Z.AI (GLM). Ele precisa de um servidor completo para funcionar, pois roda processos longos, acessa o filesystem local e executa navegadores invisiveis. Plataformas serverless como Vercel ou Netlify nao funcionam para este tipo de aplicacao.
A Oracle Cloud Always Free oferece maquinas virtuais gratuitas permanentemente (nao e trial). Com ela, voce tem um VPS real com Ubuntu, acesso root completo, filesystem persistente e sem limites de tempo de execucao. E a melhor opcao para o p3pp4.

Atencao

A Oracle pede cartao de credito na criacao da conta para verificar identidade. Nao sera cobrada enquanto voce usar os recursos do Always Free. Porem, se criar recursos pagos (como VMs AMD maiores), sera cobrado. Fique atento ao criar apenas recursos Free.

Capitulo 2
Criando a Conta na Oracle Cloud
1
Acesse o site da Oracle Cloud

Abra o navegador e va para:

https://cloud.oracle.com/free

Clique no botao "Start for Free" ou "Comece gratuitamente".

2
Preencha seus dados pessoais

Informe nome, email e pais. Use um email valido porque a Oracle enviara um codigo de verificacao. Escolha Brasil como pais (ou onde preferir). O pais define a regiao do datacenter.

3
Verifique o email

A Oracle enviara um codigo de verificacao para o email informado. Copie o codigo e cole no formulario. Este passo e obrigatorio e garante que voce e o dono do email.

4
Defina uma senha

Crie uma senha forte (minimo 12 caracteres, com maiuscula, minuscula, numero e caractere especial). Anote essa senha, voce precisara dela para acessar o painel da Oracle Cloud.

5
Adicione o cartao de credito

A Oracle exige cartao de credito para verificacao de identidade. Voce nao sera cobrado. A Oracle faz uma autorizacao temporaria de aproximadamente R$ 5 e depois estorna. Isso so acontece uma vez, para confirmar que o cartao e valido.

Cartoes pre-pagos internacionais e debito podem nao ser aceitos. Use cartao de credito se possivel.

6
Aguarde a ativacao

Apos preencher tudo, a Oracle analizara sua conta. Isso pode levar de 5 minutos a 48 horas. Na maioria dos casos, leva menos de 30 minutos. Voce recebera um email quando a conta estiver ativa.

Pronto

Quando receber o email de confirmacao, sua conta Oracle Cloud estara ativa e voce podera acessar o painel em cloud.oracle.com.

Capitulo 3
Criando a Maquina Virtual (VM)
Agora vamos criar a maquina virtual gratuita que rodara o p3pp4. Recomendamos a VM ARM (Ampere) que oferece mais recursos no plano gratuito. O processo envolve navegar pelo painel da Oracle, escolher a configuracao correta e configurar o acesso SSH.
1
Acesse o menu de Compute

No painel da Oracle Cloud, clique no icone de menu (tres linhas horizontais, canto superior esquerdo). Va em Compute > Instances.

2
Crie uma nova instancia

Clique no botao "Create Instance". A Oracle sugerira uma configuracao padrao. Voce precisara alterar alguns campos para usar o plano gratuito.

3
Configure o nome e a imagem

Name: Digite p3pp4-server (ou qualquer nome que preferir).

Image: Clique em "Edit" ao lado de Image and Shape. Escolha Canonical Ubuntu 22.04 (ou 24.04 se disponivel). Esta e a imagem mais compativel com Node.js e Playwright.

4
Escolha o Shape (tipo de VM) gratuito

Clique em "Change Shape". Selecione VM.Standard.A1.Flex (ARM Ampere). Configure:

  • OCPU: 1 (minimo necessario, voce pode escalar depois)
  • Memory: 6 GB (ou 12 GB se quiser mais folga)

Importante: O selo "Always Free eligible" deve aparecer ao lado do shape. Se nao aparecer, voce esta selecionando um recurso pago.

5
Configure a chave SSH

Na secao "Add SSH keys", escolha "Generate a key pair". A Oracle gerara duas chaves (publica e privada). Clique em "Save Private Key" e guarde o arquivo .key com seguranca. Essa e a unica vez que podera baixar essa chave.

Se preferir, pode usar uma chave SSH propria selecionando "Upload public key" e colando sua chave publica.

6
Configure o Boot Volume

O Boot Volume padrao de 47 GB e suficiente para o p3pp4. Se quiser mais espaco, clique em "Specify a custom boot volume size" e aumente ate 200 GB (limite gratuito). Nao altere outras opcoes.

7
Crie a instancia

Clique em "Create" no final da pagina. A Oracle criara a VM em poucos minutos. Quando o status mudar de "Provisioning" para "Running", sua VM estara pronta.

Anote o Public IP Address que aparece na pagina da instancia. Voce precisara dele para conectar via SSH.

VM Pronta

Sua VM esta rodando. Anote o IP publico e o caminho da chave SSH privada. Agora vamos conectar e configurar.

Capitulo 4
Conectando via SSH
Para configurar a VM, voce precisa conectar a ela remotamente via SSH. O SSH e um protocolo seguro de acesso remoto. Voce usara o terminal do seu computador para enviar comandos para a VM na Oracle Cloud. O processo e simples, mas requer que a chave privada esteja com as permissoes corretas.
1
Abra o terminal no seu computador

No Windows, abra o PowerShell ou Terminal. No Mac ou Linux, abra o Terminal nativo.

2
Ajuste as permissoes da chave SSH

A chave privada so funciona com permissoes restritas. Execute:

chmod 400 ~/Downloads/ssh-key-*.key

Substitua o caminho pelo local onde salvou a chave privada no passo anterior.

3
Conecte na VM

Execute o comando abaixo, substituindo o IP e o caminho da chave:

ssh -i ~/Downloads/ssh-key-*.key ubuntu@SEU_IP_PUBLICO

O usuario padrao da VM Ubuntu na Oracle Cloud e ubuntu. Na primeira conexao, digite yes quando perguntado sobre o fingerprint.

Conectado

Se voce vir o prompt ubuntu@p3pp4-server:~$, esta conectado com sucesso na sua VM!

Capitulo 5
Instalando as Dependencias
Agora vamos instalar tudo que o p3pp4 precisa: Node.js, Python, Playwright, Prisma e as ferramentas do sistema. Execute cada comando um por vez no terminal conectado a VM. Este processo leva aproximadamente 10 a 15 minutos dependendo da velocidade da conexao.
1
Atualize o sistema
sudo apt update && sudo apt upgrade -y

Este comando atualiza todos os pacotes do Ubuntu. Pode levar alguns minutos na primeira vez.

2
Instale ferramentas basicas
sudo apt install -y curl git wget unzip build-essential

Essas ferramentas sao necessarias para compilar pacotes nativos (como sharp e Prisma) e baixar outros instaladores.

3
Instale o Node.js 20
curl -fsSL https://deb.nodesource.com/setup_20.x | sudo -E bash - sudo apt install -y nodejs

Isso instala o Node.js 20 LTS e o npm. Verifique com node -v e npm -v.

4
Instale o Python 3 e pip
sudo apt install -y python3 python3-pip python3-venv

O solver do p3pp4 roda em Python. Verifique com python3 --version.

5
Instale o Playwright (navegador headless)
sudo npx playwright install --with-deps chromium

Este comando instala o navegador Chromium e todas as dependencias do sistema que o Playwright precisa. E o passo mais demorado (pode levar 3 a 5 minutos). O argumento --with-deps instala automaticamente as bibliotecas nativas do Ubuntu que o Chromium precisa.

6
Instale o z-ai CLI
sudo npm install -g z-ai-web-dev-sdk

Este e o SDK de IA utilizado pelo solver para analise visual e calculos. Apos a instalacao, verifique com z-ai --version.

7
Instale dependencias do Playwright para Python
pip3 install playwright python3 -m playwright install chromium

O solver Python tambem usa Playwright diretamente. Isso instala o binding Python e o navegador Chromium para ele.

Capitulo 6
Fazendo o Deploy do p3pp4
Agora vamos clonar o repositorio do p3pp4 do GitHub, instalar as dependencias Node.js, configurar o banco de dados e rodar o projeto. Este e o passo principal onde o seu projeto fica online.
1
Clone o repositorio
cd ~ git clone https://github.com/SrBatat/p3pp0.git p3pp4 cd p3pp4

Isso baixa todo o codigo do seu repositorio GitHub para a VM.

2
Instale as dependencias Node.js
npm install

Este comando instala todas as bibliotecas listadas no package.json. Pode levar 2 a 5 minutos porque compila pacotes nativos como sharp e Prisma.

3
Configure as variaveis de ambiente
cp .env.example .env

Depois edite o .env para o caminho correto do banco:

DATABASE_URL="file:./db/custom.db"

O arquivo .env nao esta no GitHub (esta no .gitignore), entao voce precisa cria-lo manualmente na VM a partir do template .env.example.

4
Crie o diretorio do banco e gere o Prisma Client
mkdir -p db npx prisma generate npx prisma db push

O prisma generate cria o client do Prisma. O prisma db push cria o arquivo SQLite do banco de dados. Sem isso, a aplicacao falha ao tentar acessar o banco.

5
Crie os diretorios necessarios
mkdir -p download/physics-bot/screenshots mkdir -p upload

Esses diretorios sao usados pelo solver para salvar screenshots e arquivos temporarios. Eles estao no .gitignore, entao nao vem do GitHub.

6
Adapte os caminhos do projeto

Os caminhos hardcoded /home/z/my-project precisam ser alterados para /home/ubuntu/p3pp4. Edite os seguintes arquivos na VM:

# No arquivo scripts/physics_bot.py: # Altere a linha: OUTPUT_DIR = Path("/home/z/my-project/download/physics-bot") # Para: OUTPUT_DIR = Path("/home/ubuntu/p3pp4/download/physics-bot")
# No arquivo src/app/api/solve/start/route.ts: # Altere todas as ocorrencias de: /home/z/my-project # Para: /home/ubuntu/p3pp4

Para editar rapidamente, use o comando nano:

nano scripts/physics_bot.py # Ctrl+W para buscar, Ctrl+O para salvar, Ctrl+X para sair
7
Remova output: standalone do next.config.ts

Edite o arquivo next.config.ts e remova a linha output: "standalone". Deixe assim:

const nextConfig: NextConfig = { typescript: { ignoreBuildErrors: true, }, reactStrictMode: false, };

O modo standalone e para Docker. Para rodar diretamente na VM, usaremos o modo padrao do Next.js.

8
Build do projeto
npm run build

Este comando compila o Next.js. Se der erro de TypeScript, verifique se ignoreBuildErrors: true esta no next.config.ts. O build pode levar 1 a 3 minutos.

9
Rode o projeto
npm run start &

O & no final roda o processo em background. O servidor estara disponivel em http://SEU_IP_PUBLICO:3000. Porem, voce ainda precisa abrir a porta 3000 no firewall (veja o proximo capitulo).

Capitulo 7
Configurando o Firewall e Acesso
A Oracle Cloud tem dois firewalls: um no nivel da rede (Security List / Network Security Group) e outro no nivel da VM (iptables). Voce precisa liberar a porta 3000 em ambos para conseguir acessar o p3pp4 pelo navegador. Alem disso, para producao, e recomendado configurar o Caddy como proxy reverso na porta 80/443 com HTTPS automatico.
1
Libere a porta no Security List (Oracle Cloud)

No painel da Oracle Cloud, va em Networking > Virtual Cloud Networks. Clique no VCN que foi criado automaticamente com a VM. Va em Security Lists e clique no security list padrao.

Clique em "Add Ingress Rules" e preencha:

  • Source Type: CIDR
  • Source CIDR: 0.0.0.0/0
  • IP Protocol: TCP
  • Destination Port Range: 3000

Clique em "Add Ingress Rules". Se quiser usar Caddy na porta 80/443 depois, libere tambem as portas 80,443.

2
Libere a porta no iptables (dentro da VM)

A Oracle Cloud configura iptables restritivo por padrao. Execute na VM:

sudo iptables -I INPUT 6 -m state --state NEW -p tcp --dport 3000 -j ACCEPT sudo netfilter-persistent save

Se der erro no netfilter-persistent, instale com:

sudo apt install -y iptables-persistent
3
Teste o acesso

Abra o navegador no seu computador e acesse:

http://SEU_IP_PUBLICO:3000

Se tudo estiver correto, voce vera a interface gótica dark do p3pp4!

Capitulo 8
HTTPS com Caddy (Opcional)
Para ter HTTPS automatico e um dominio personalizado, instale o Caddy. Ele funciona como proxy reverso: recebe requisicoes na porta 80/443 e redireciona para o Next.js na porta 3000. O Caddy gera certificado SSL automaticamente via Let's Encrypt, sem nenhuma configuracao manual.
1
Instale o Caddy
sudo apt install -y debian-keyring debian-archive-keyring apt-transport-https curl curl -1sLf 'https://dl.cloudsmith.io/public/caddy/stable/gpg.key' | sudo gpg --dearmor -o /usr/share/keyrings/caddy-stable-archive-keyring.gpg curl -1sLf 'https://dl.cloudsmith.io/public/caddy/stable/debian.deb.txt' | sudo tee /etc/apt/sources.list.d/caddy-stable.list sudo apt update sudo apt install caddy
2
Configure o Caddyfile

Se voce tem um dominio (ex: p3pp4.com.br), aponte o DNS A para o IP da VM. Depois edite o Caddyfile:

sudo nano /etc/caddy/Caddyfile

Substitua o conteudo por:

# Com dominio (HTTPS automatico): p3pp4.com.br { reverse_proxy localhost:3000 } # Sem dominio (so HTTP): :80 { reverse_proxy localhost:3000 }
3
Reinicie o Caddy
sudo systemctl restart caddy

Se estiver usando dominio, acesse https://p3pp4.com.br. O Caddy obtera automaticamente o certificado SSL. Se estiver sem dominio, acesse http://SEU_IP_PUBLICO.

Lembrete

Se usar dominio, libere as portas 80 e 443 no Security List e no iptables da mesma forma que abriu a porta 3000. O Caddy precisa dessas portas para HTTPS funcionar.

Capitulo 9
Mantendo o Servidor Rodando
Se voce rodar npm run start manualmente, o servidor morre quando voce fecha o terminal. Para manter o p3pp4 rodando permanentemente (mesmo apos reiniciar a VM), use o systemd — o gerenciador de servicos padrao do Ubuntu. Ele reinicia automaticamente o p3pp4 se ele travar e inicia junto com o boot da VM.
1
Crie o servico do systemd
sudo nano /etc/systemd/system/p3pp4.service

Cole o seguinte conteudo:

[Unit] Description=p3pp4 Physics Solver After=network.target [Service] Type=simple User=ubuntu WorkingDirectory=/home/ubuntu/p3pp4 ExecStart=/usr/bin/npm run start Restart=always RestartSec=5 Environment=NODE_ENV=production [Install] WantedBy=multi-user.target
2
Ative e inicie o servico
sudo systemctl daemon-reload sudo systemctl enable p3pp4 sudo systemctl start p3pp4

O enable faz o p3pp4 iniciar automaticamente quando a VM reiniciar. O start inicia agora.

3
Verifique o status
sudo systemctl status p3pp4

Deve mostrar "active (running)". Se mostrar erro, verifique os logs:

sudo journalctl -u p3pp4 -f
Servidor Permanente

Agora o p3pp4 roda 24/7, reinicia automaticamente se travar, e sobe junto com a VM. Voce pode fechar o terminal sem preocupacao.

Capitulo 11
Resolucao de Problemas
Se algo nao funcionar, verifique estes problemas comuns. A maioria dos erros acontece nas primeiras configuracoes e sao faceis de resolver. Sempre verifique os logs primeiro com sudo journalctl -u p3pp4 -f para identificar a causa real do erro.
ProblemaCausa ProvavelSolucao
Nao consigo acessar http://IP:3000 Firewall bloqueando Verifique Security List + iptables. Veja Capitulo 7.
Erro "Cannot find module" Dependencias faltando Rode npm install novamente.
Erro Prisma "P1001" Banco nao existe Rode npx prisma db push.
Playwright falha "Browser not found" Chromium nao instalado Rode sudo npx playwright install --with-deps chromium.
Solver retorna erro 502 Timeout do processo A arquitetura assincrona ja resolve isso. Verifique se o Python esta funcionando com python3 scripts/physics_bot.py.
VM nao responde Instancia parada No painel Oracle, va em Compute > Instances e clique em Start.
z-ai: comando nao encontrado SDK nao instalado globalmente Rode sudo npm install -g z-ai-web-dev-sdk.
Cadastro Oracle rejeitado Cartao nao aceito Tente outro cartao ou contate o suporte Oracle.
Dica Importante

A Oracle Cloud pode reciclar (parar) VMs gratuitas que ficam muito tempo sem uso. Se sua VM parar, basta iniciar novamente pelo painel. Para evitar isso, configure um ping automatico ou um cronjob que mantenha a VM ativa.
